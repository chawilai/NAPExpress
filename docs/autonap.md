# AutoNAP — คู่มือการใช้งาน

## ภาพรวม

AutoNAP คือระบบบันทึกข้อมูลเข้าเว็บ NAP Plus ของ สปสช. โดยอัตโนมัติ แทนการให้เจ้าหน้าที่นั่งคีย์ข้อมูลซ้ำด้วยมือ

**ปัญหาเดิม:** ข้อมูลอยู่ใน CAREMAT ครบหมดแล้ว แต่ต้องจ้างเจ้าหน้าที่มาคีย์ซ้ำเข้าเว็บ สปสช. เพื่อเคลมเงินคืน

**วิธีแก้:** CAREMAT ส่งข้อมูลไปยัง AutoNAP → ระบบเปิดเว็บ สปสช. แล้วกรอกฟอร์มให้อัตโนมัติ → แจ้งผลกลับมาแบบ realtime

## 1. AutoNAP Reach RR

### ขั้นตอนการใช้งาน

1. เปิดหน้า **NHSO → Reach To NAP**
2. เลือกช่วงเวลา กดดึงข้อมูล
3. กดปุ่ม **"เลือกรายการที่ยังไม่บันทึก NAP"**
4. กรองข้อมูลตามต้องการ (วันที่, ที่มา, KP, CBS, Location)
5. เลือกเคสที่ต้องการ → กดปุ่ม **"ส่งบันทึก AutoNAP"**
6. แสดง QR Code → เจ้าหน้าที่สแกนด้วยแอป **ThaID** เพื่อยืนยันตัวตน
7. ระบบกรอกฟอร์ม RR ทีละเคส → แสดงความคืบหน้าแบบ realtime
8. เสร็จ → รหัส NAP อัพเดตในตารางอัตโนมัติ

### ข้อมูลที่ระบบกรอกให้

| ข้อมูล | ตัวอย่าง |
|--------|----------|
| เลขบัตรประชาชน | 1409900584494 |
| UIC | วห120832 |
| วันที่เข้าถึง (RR) | 28/03/2569 |
| กลุ่มเสี่ยง | MSM / TG / FSW ฯลฯ |
| อาชีพ | รับจ้าง |
| ถุงยาง + สารหล่อลื่น | จำนวนที่จ่าย |
| สถานที่ / ช่องทาง / แหล่งเงิน | ตาม site |

### ผลลัพธ์

- สำเร็จ → ได้รหัส **RR code** เช่น `RR-2569-123456`
- Error → แสดงข้อความ error สามารถส่งใหม่ได้

## 2. AutoNAP VCT + Request Lab HIV + HIV Test Result

### ขั้นตอนการใช้งาน

1. เปิดหน้า **NHSO → Clinic HIV To NAP**
2. เลือกช่วงเวลา กดดึงข้อมูล
3. กดปุ่ม **"เลือกรายการที่ยังไม่บันทึก VCT"**
4. กรองข้อมูลตามต้องการ (วันที่, Source, KP, CBS, **ผลเลือด HIV**)
5. เลือกเคสที่ต้องการ (ต้องเลือกเอง ไม่ได้เลือกให้อัตโนมัติ)
6. เลือกโหมด:
   - ✅ **"บันทึก VCT + Request Lab HIV"** (ค่าเริ่มต้น) → บันทึก VCT แล้วส่งตรวจ HIV + ลงผลต่อ
   - ☐ **"บันทึกแค่ VCT อย่างเดียว"** → บันทึกเฉพาะ VCT
7. กดปุ่ม **"ส่งบันทึก AutoNAP VCT"**
8. แสดง QR Code → เจ้าหน้าที่สแกนด้วยแอป **ThaID**
9. ระบบกรอกฟอร์มทีละเคส:
   - **ขั้นที่ 1:** บันทึก VCT → ได้รหัส VCT ID
   - **ขั้นที่ 2:** บันทึก Request Lab HIV → ได้รหัส ANTIHIV
   - **ขั้นที่ 3:** ลงผลตรวจ HIV → บันทึก Negative/Positive/Inconclusive
10. เสร็จ → VCT checkbox ถูก check + รหัส NAP อัพเดตในตาราง

### ข้อมูลที่ระบบใช้

| ข้อมูล | ตัวอย่าง |
|--------|----------|
| เลขบัตรประชาชน (13 หลัก) | 1550700153989 |
| กลุ่มเป้าหมาย (KP) | MSM |
| ผู้ให้บริการ (CBS) | นภสร ใจกาศ |
| วันที่ให้บริการ | 04/03/2569 |
| UIC | ศอ160742 |
| Source | walk in / reach |
| ผล HIV | Negative / Positive / Inconclusive |
| ผล Syphilis | N (Negative) / R (Reactive) |
| ผล CT (Chlamydia) | CT Detected / CT Not Detected |
| ผล NG (Gonorrhea) | NG Detected / NG Not Detected |

### ข้อมูลที่ AutoNAP กรอกให้อัตโนมัติ

CAREMAT ส่งแค่ข้อมูลพื้นฐาน ส่วนรายละเอียดฟอร์มระบบกรอกตามหลักเกณฑ์:

| ข้อมูลในฟอร์ม สปสช. | ระบบกรอกว่า |
|---------------------|------------|
| ช่องทางมารับบริการ | การเข้าถึงโดยงบป้องกันเอดส์ (RRTTR) |
| หน่วยงาน/แผนกที่ส่งต่อ | walk in → "หน่วยบริการ หรือ คลินิกภายในรพ" / reach → "การออกพื้นที่โดย หน่วยบริการ" |
| ประเมินความเสี่ยง | มีพฤติกรรมเสี่ยง |
| กลุ่มผู้มารับบริการ | ตาม KP ที่ส่งมา |
| ปัจจัยเสี่ยง | มีเพศสัมพันธ์โดยไม่ป้องกัน (ถ้า PWID → เพิ่ม "ใช้ยาเสพติด") |
| Pre-test | ทำ, **CITC** (ผู้รับบริการแสดงความต้องการตรวจด้วยตนเอง), รายบุคคล |
| Post-test | ทำ, วันเดียวกับวันที่ให้บริการ |
| Couple counseling | **มีคู่ แต่คู่ไม่ได้ตรวจ** |
| การส่งต่อ STI | ตามผลตรวจ (ดูตารางด้านล่าง) |
| เมทาโดน | ไม่ได้รับ |
| ถุงยางอนามัย | รับ, ทุกขนาด (49, 52, 53, 54, 56mm) = 10 ชิ้น/ขนาด, สารหล่อลื่น = 10 ซอง |

### การส่งต่อ STI (กรอกอัตโนมัติตามผลตรวจ)

| เงื่อนไข | ระบบกรอกว่า |
|----------|------------|
| **มีผลตรวจ STI อย่างน้อย 1 รายการ** | ส่งต่อ → ได้รับการตรวจ |
| ผล Syphilis เป็น N หรือ R | เลือก "ซิฟิลิส (Syphilis)" |
| ผล Syphilis เป็น R (บวก) | เลือก "ผลบวกและตรวจยืนยันด้วย TPHA" เพิ่ม |
| ผล CT มีค่า | เลือก "หนองในเทียม (Chlamydia)" |
| ผล NG มีค่า | เลือก "หนองใน (Gonorrhea)" |
| **ไม่มีผลตรวจ STI เลย** | **ส่งต่อ → ไม่ได้ตรวจ** |

### VCT ซ้ำ — ดึงค่าเดิมอัตโนมัติ

เมื่อ สปสช. แจ้งว่า "VCT ซ้ำ" ระบบจะ **ค้นหา code เดิม** แทนที่จะ error:

| กรณี | สิ่งที่ระบบทำ | Comment ใน Callback |
|------|-------------|---------------------|
| มี VCT + Lab + ผลตรวจ ครบ | ดึง code เดิมทั้ง 3 ส่ง callback กลับ | "VCT,Lab,Result ครบ ดึงค่าเดิม" |
| มี VCT + Lab แต่ยังไม่ลงผล | ดึง VCT + Lab กลับ → ลงผลตรวจต่อ | "VCT, Lab ซ้ำ — ดึงค่าเดิม" |
| มี VCT แต่ยังไม่มี Lab | ดึง VCT กลับ → ทำ Lab + ลงผลต่อ | "VCT ซ้ำ — ดึงค่าเดิม" |

## วิธีใช้งาน (สรุป)

### เปิดหน้า

| ฟอร์ม | เมนู |
|-------|------|
| Reach RR | NHSO → Reach To NAP |
| VCT + Lab HIV + ผลตรวจ | NHSO → Clinic HIV To NAP |

### ขั้นตอนหลัก

1. เลือกช่วงเวลา → กดดึงข้อมูล
2. กดปุ่ม **"เลือกรายการที่ยังไม่บันทึก"** → ระบบสแกนทุกรายการหาเคสที่ยังไม่ครบ
3. ใช้ตัวกรอง (Source, KP, CBS, HIV) → กด **"เลือกทั้งหมด"** = เลือกเฉพาะรายการที่กรองอยู่
4. กดส่ง → สแกน QR ThaID → รอระบบทำงาน
5. ดูความคืบหน้าแบบ realtime → เสร็จแล้วตรวจสอบรหัสในตาราง

### การดู Progress ระหว่างทำงาน

- ถ้าปิดหน้าไประหว่าง job ทำงาน → กลับมาเปิดใหม่ได้
- CAREMAT เรียก `GET /api/jobs/status?site={site}&form_type=VCT` เพื่อเช็คว่ามี job ทำงานอยู่ไหม
- ถ้ามี → กดปุ่ม "ดู Progress" → reconnect เข้า Ably channel เดิม → เห็น event ใหม่จากจุดที่ reconnect

### การดูผลลัพธ์

- **สำเร็จ** → รหัส NAP แสดงในตาราง + checkbox ถูก check อัตโนมัติ
- **Error** → hover บน comment เห็นข้อความเต็ม → สามารถส่งใหม่ได้

### Email Report

ทุก job ที่จบจะส่ง email report อัตโนมัติไปยัง admin ประกอบด้วย:
- Site, Form Type, NAP Username, NAP Site
- เวลาเริ่ม/จบ, เฉลี่ยต่อรายการ
- สรุป Success/Failed + ตารางรายละเอียดทุก record

## staff_name — ชื่อเจ้าหน้าที่ผู้บันทึก

CAREMAT ส่ง `staff_name` มาใน request (ชื่อผู้ login CAREMAT) → ระบบใช้เป็น `nap_staff` ใน callback กลับไปยัง CAREMAT เพื่อบันทึกว่าใครเป็นคนสั่งบันทึก

## Queue Status — คิวรอส่ง

เมื่อ CAREMAT ส่ง request มา ถ้ามี job อื่นอยู่ในคิว ระบบจะตอบกลับพร้อมสถานะคิว:

```json
{
  "status": "ok",
  "job_id": "autonap-abc123",
  "queued": true,
  "estimated_wait_minutes": 12,
  "message": "Job dispatched — 30 items queued for VCT (ThaiID)"
}
```

| Field | คำอธิบาย |
|-------|---------|
| `queued` | `true` = มี job อื่นรออยู่ข้างหน้า, `false` = ทำงานได้ทันที |
| `estimated_wait_minutes` | เวลาโดยประมาณที่ต้องรอ (นาที) |

## Browser Crash Recovery

ถ้า browser crash ระหว่างทำงาน ระบบจะ:
1. ตรวจจับว่า browser หยุดทำงาน
2. เปิด browser ใหม่อัตโนมัติ + inject cookies เดิม
3. retry record ที่ค้าง (สูงสุด **2 ครั้ง/record**)
4. ถ้า retry 2 ครั้งแล้วยัง crash → mark error แล้วข้ามไป record ถัดไป

### Batch Browser Restart

เพื่อป้องกัน memory leak ทุก **25 records** ระบบจะ:
- ปิด browser ปัจจุบัน
- เปิด browser ใหม่ + inject cookies
- ทำงานต่อจาก record ถัดไป

## Dashboard — หน้าติดตามงาน

| URL | คำอธิบาย |
|-----|---------|
| `/autonap` | หน้า Dashboard สำหรับดู job ทั้งหมด |

### ความสามารถ

- แสดง job ทั้งหมดพร้อมสถานะ (รอคิว, กำลังทำ, สำเร็จ, ล้มเหลว)
- ติดตามความคืบหน้าแบบ real-time ผ่าน Ably
- ดูรายละเอียด record แต่ละรายการ (NAP code, error)

### Login Timeout — ไม่มี record ใน Dashboard

ถ้า ThaiD login ไม่สำเร็จ (หมดเวลา 60 วินาที):
- ไม่ส่ง email report
- ไม่มี record ใน dashboard
- CAREMAT ได้รับ error callback เฉพาะ Ably event `job:login:failed`

## ข้อควรระวัง

1. **ส่งได้สูงสุด 50 เคส/ครั้ง** — ใช้เวลาประมาณ 15-25 นาที
2. **ThaID จำเป็น** — เว็บ สปสช. ต้องยืนยันตัวตนผ่าน ThaID ทุกครั้ง
3. **QR 60 วินาที** — ต้องสแกนให้ทัน ถ้าไม่ทันต้องส่งใหม่
4. **ตรวจสอบรหัส NAP** — หลังส่งเสร็จ ตรวจสอบว่าได้รหัสครบทุกเคส
5. **ส่งใหม่ได้** — เคสที่ error สามารถส่งใหม่ได้ ระบบจะไม่ซ้ำกับเคสที่มีรหัสแล้ว
6. **VCT ซ้ำ** → ระบบจะค้นหา code เดิมส่งกลับอัตโนมัติ ไม่ต้องคีย์ใหม่
7. **ผลเลือด Positive** — ควรตรวจสอบก่อนส่ง ใช้ตัวกรองผล HIV เพื่อคัดกรอง
8. **ผู้บันทึก NAP** — ใช้ `staff_name` ที่ CAREMAT ส่งมา (ชื่อผู้ login CAREMAT) เป็น `nap_staff` ใน callback
9. **ข้อความ Error** — แสดงเป็นภาษาไทยที่อ่านเข้าใจง่าย (log เก็บรายละเอียดทางเทคนิคสำหรับ debug)
10. **Login timeout** — ถ้า ThaiD login ไม่สำเร็จ (หมดเวลา 60 วินาที) → ระบบจะไม่ส่ง email report และไม่มี record ใน dashboard
11. **Browser crash** — ถ้า browser crash ระหว่างทำงาน ระบบจะ restart browser อัตโนมัติและ retry record นั้น (สูงสุด 2 ครั้ง/record)
12. **Batch restart** — ทุก 25 records ระบบจะปิด browser แล้วเปิดใหม่เพื่อป้องกัน memory leak
