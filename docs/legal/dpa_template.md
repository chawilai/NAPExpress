# ข้อตกลงการประมวลผลข้อมูลส่วนบุคคล (Data Processing Agreement — DPA)

**ระหว่าง**
- **[ชื่อผู้ควบคุมข้อมูล]** ("ลูกค้า") — หน่วยบริการสุขภาพที่ขึ้นทะเบียนกับ สปสช.
- **[ชื่อนิติบุคคล AutoNAP]** ("ผู้ประมวลผล") — ผู้ให้บริการระบบ AutoNAP

> ⚖️ **Disclaimer:** เอกสารนี้เป็น**template** สำหรับใช้เป็นจุดเริ่มต้น ควรให้ทนายของทั้งสองฝ่าย review และปรับแก้ก่อนลงนาม

## 1. คำจำกัดความ

ในสัญญานี้ คำต่อไปนี้มีความหมายตามที่กำหนด เว้นแต่ context จะบ่งชี้เป็นอย่างอื่น:

| คำ | ความหมาย |
|---|---|
| **PDPA** | พระราชบัญญัติคุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 |
| **ข้อมูลส่วนบุคคล** | ข้อมูลตามนิยามใน PDPA มาตรา 6 |
| **ข้อมูลอ่อนไหว** | ข้อมูลส่วนบุคคลตามมาตรา 26 (เช่น ข้อมูลสุขภาพ, เชื้อชาติ, รสนิยมทางเพศ) |
| **Data Controller / ผู้ควบคุมข้อมูล** | "ลูกค้า" — ตัดสินใจวัตถุประสงค์และวิธีประมวลผล |
| **Data Processor / ผู้ประมวลผล** | "AutoNAP" — ประมวลผลตามคำสั่งของ Data Controller |
| **Data Subject / เจ้าของข้อมูล** | ผู้รับบริการของลูกค้า (ผู้ป่วย/ลูกค้าคลินิก) |
| **Processing / การประมวลผล** | การเก็บ, ใช้, เปิดเผย, ลบ, หรือดำเนินการกับข้อมูล |
| **บริการ** | ระบบ AutoNAP ที่ช่วยบันทึกข้อมูลเข้าระบบ NAP Plus ของ สปสช. อัตโนมัติ |

---

## 2. บทบาทและความสัมพันธ์

2.1 ลูกค้าเป็น **ผู้ควบคุมข้อมูล** (Data Controller) ตามความหมายของ PDPA สำหรับข้อมูลส่วนบุคคลที่ส่งให้ AutoNAP ประมวลผล

2.2 AutoNAP เป็น **ผู้ประมวลผลข้อมูล** (Data Processor) ซึ่งดำเนินการประมวลผลตามคำสั่งและในนามของลูกค้าเท่านั้น

2.3 AutoNAP **จะไม่ใช้ข้อมูล** เพื่อวัตถุประสงค์ของตัวเอง ไม่ว่าในรูปแบบใด รวมถึง:
- ไม่ใช้เพื่อการตลาด
- ไม่ขายข้อมูลให้บุคคลที่สาม
- ไม่ใช้ train model / AI
- ไม่ใช้วิเคราะห์เชิง aggregate เพื่อเผยแพร่

---

## 3. ประเภทและขอบเขตของข้อมูล

### 3.1 ข้อมูลที่ลูกค้าส่งให้ประมวลผล (ชั่วคราว)

AutoNAP จะประมวลผลข้อมูลต่อไปนี้ **ในหน่วยความจำ (in-memory) เท่านั้น** ในระหว่างการประมวลผลแต่ละ job:

- เลขประจำตัวประชาชน (PID) 13 หลัก
- UIC (รหัสผู้รับบริการของคลินิก)
- วันที่ให้บริการ
- Key Population (KP)
- อาชีพ
- ข้อมูลในฟอร์ม NAP (risk behavior, condoms, lab results เมื่อเกี่ยวข้อง)

### 3.2 ข้อมูลที่ AutoNAP เก็บรักษา (Persistent storage)

หลังการประมวลผลเสร็จสิ้น AutoNAP **เก็บเฉพาะ** ข้อมูลต่อไปนี้เพื่อวัตถุประสงค์ของการตรวจสอบ (audit):

| ฟิลด์ | ตัวอย่าง | หมายเหตุ |
|---|---|---|
| PID masked | `xxxx1234` | เฉพาะ 4 ตัวท้าย |
| NAP code | `RR-2569-123456` | รหัสที่ได้จาก สปสช. |
| Form type | `RR` / `VCT` / `Lab` / `Result` | |
| Success flag | true/false | |
| Timestamp | 2026-04-10 10:30:15 | |

AutoNAP **จะไม่เก็บ**:
- ❌ PID เต็ม 13 หลัก
- ❌ ชื่อ-นามสกุล
- ❌ เบอร์โทรศัพท์
- ❌ ที่อยู่
- ❌ อีเมล
- ❌ ข้อมูลการแพทย์รายละเอียด
- ❌ ประวัติการรักษา
- ❌ CSV/Excel ไฟล์ที่ลูกค้า upload (ลบหลังประมวลผล)

### 3.3 ระยะเวลาเก็บรักษา

- **ข้อมูล audit log** เก็บไว้ไม่เกิน **90 วัน** นับจากวันที่ประมวลผล หลังจากนั้นระบบจะลบอัตโนมัติ
- ลูกค้าสามารถขอให้ลบก่อนกำหนดได้ตลอดเวลา (ภายใน 7 วันทำการ)

## 4. วัตถุประสงค์และขอบเขตการประมวลผล

4.1 **วัตถุประสงค์เดียว:** การประมวลผลข้อมูลมีวัตถุประสงค์**เฉพาะ**เพื่อบันทึกข้อมูลเข้าระบบ NAP Plus ของ สปสช. ตามภารกิจทางกฎหมายของลูกค้าในฐานะหน่วยบริการ สปสช.

4.2 **Processing operations:**
- รับข้อมูลจากลูกค้า (CSV upload / API)
- แปลง/จัดรูปข้อมูลให้ตรงกับฟอร์ม NAP
- เข้าสู่ระบบ NAP Plus ผ่าน ThaiID ของผู้ใช้งาน หรือ credentials ของลูกค้า
- ส่งข้อมูลเข้าฟอร์ม NAP
- บันทึกผลลัพธ์ (NAP code + masked PID) สำหรับ audit
- ส่ง callback กลับระบบของลูกค้า (ถ้ามี)
- ลบข้อมูลเคสจากหน่วยความจำหลังประมวลผลเสร็จ

4.3 **ข้อจำกัด:** AutoNAP จะไม่ประมวลผลข้อมูลนอกขอบเขตนี้ เว้นแต่ได้รับคำสั่งเป็นลายลักษณ์อักษรจากลูกค้า

---

## 5. มาตรการรักษาความมั่นคงปลอดภัย

5.1 AutoNAP จะใช้มาตรการทางเทคนิคและองค์กรที่เหมาะสมเพื่อรักษาความปลอดภัยของข้อมูล ได้แก่:

**มาตรการทางเทคนิค:**
- (a) การเข้ารหัสข้อมูลระหว่างส่ง (TLS 1.2+)
- (b) การเข้ารหัสข้อมูลที่เก็บรักษา (encryption at rest)
- (c) การควบคุมการเข้าถึง (access control) — role-based
- (d) การบันทึกการใช้งาน (access log / audit log)
- (e) การสำรองข้อมูล (backup) พร้อมการเข้ารหัส
- (f) การอัพเดทระบบ (patching) เป็นประจำ
- (g) **ไม่เก็บข้อมูลเคสในระบบ** (data minimization by design)

**มาตรการทางองค์กร:**
- (a) การควบคุมการเข้าถึงเฉพาะบุคคลที่ได้รับอนุญาต
- (b) การฝึกอบรม PDPA ให้พนักงาน
- (c) NDA (non-disclosure agreement) กับพนักงานและ contractor
- (d) การ review ระบบ security ประจำปี

5.2 **Data localization:** ข้อมูลทั้งหมดเก็บอยู่บน server ที่ตั้งอยู่ในประเทศไทย AutoNAP จะไม่ถ่ายโอนข้อมูลออกนอกประเทศโดยไม่ได้รับความยินยอมจากลูกค้าเป็นลายลักษณ์อักษรล่วงหน้า

## 6. Sub-Processors (ผู้ประมวลผลรอง)

6.1 AutoNAP อาจใช้ sub-processor เพื่อให้บริการ เช่น:

| Sub-Processor | บริการ | ที่ตั้งข้อมูล |
|---|---|---|
| [VPS provider — ระบุชื่อ] | Infrastructure hosting | Thailand |
| [Email service] | ส่ง report email | Thailand/SE Asia |
| [Realtime service — Ably/Pusher] | Progress updates | Global CDN (no PII) |

6.2 **การแจ้ง:** AutoNAP จะแจ้งลูกค้าล่วงหน้าอย่างน้อย 30 วันเมื่อมีการเปลี่ยนแปลง sub-processor ลูกค้ามีสิทธิคัดค้านภายใน 14 วัน หากคัดค้าน AutoNAP จะพยายามหาทางเลือก หากไม่ได้ ลูกค้าสามารถยกเลิกสัญญาได้โดยไม่มีค่าปรับ

6.3 AutoNAP จะให้ sub-processor ทุกรายมีข้อตกลงที่**ไม่ต่ำกว่า**มาตรฐานใน DPA ฉบับนี้

## 7. หน้าที่ของ AutoNAP

7.1 **Confidentiality:** บุคคลใดที่ AutoNAP มอบหมายให้เข้าถึงข้อมูลของลูกค้าจะต้องมีข้อผูกพันรักษาความลับ (contractual หรือ legal obligation)

7.2 **ปฏิบัติตามคำสั่ง:** ประมวลผลข้อมูลตามคำสั่งของลูกค้าเท่านั้น หาก AutoNAP เห็นว่าคำสั่งใดขัดต่อ PDPA จะแจ้งลูกค้าทันที

7.3 **ช่วยเหลือลูกค้า** ในการปฏิบัติตามคำขอจาก data subject:
- คำขอเข้าถึงข้อมูล (right of access)
- คำขอแก้ไขข้อมูล (right to rectification)
- คำขอลบข้อมูล (right to erasure / right to be forgotten)
- คำขอระงับการประมวลผล
- คำขอโอนย้ายข้อมูล (right to data portability)
- คำขอคัดค้านการประมวลผล

7.4 **ช่วยเหลือลูกค้า** ในการปฏิบัติตาม:
- Privacy Impact Assessment (PIA)
- Breach notification
- Consultation with regulator (PDPC)

7.5 **Breach notification:** AutoNAP จะแจ้งลูกค้าภายใน **72 ชั่วโมง** หากพบว่ามีเหตุการณ์รั่วไหลของข้อมูล โดยระบุ:
- ลักษณะของเหตุการณ์
- ประเภทและจำนวนข้อมูลที่ได้รับผลกระทบ
- ผลกระทบที่อาจเกิดขึ้น
- มาตรการที่ดำเนินการแล้ว
- ข้อเสนอแนะเพิ่มเติม

## 8. หน้าที่ของลูกค้า (Data Controller)

8.1 **Legal basis:** ลูกค้ารับผิดชอบในการมี**ฐานทางกฎหมาย** (legal basis) ที่ถูกต้องสำหรับการประมวลผลข้อมูล เช่น:
- ความยินยอม (consent) จากผู้รับบริการ
- การปฏิบัติตามสัญญา (contract)
- ภารกิจทางกฎหมาย (legal obligation) — การรายงานต่อ สปสช.
- ประโยชน์อันชอบด้วยกฎหมาย (legitimate interest)

8.2 **Privacy notice:** ลูกค้าจะแจ้งผู้รับบริการเกี่ยวกับการใช้ผู้ประมวลผลภายนอก (AutoNAP) ใน privacy notice หรือ consent form

8.3 **Accurate instructions:** ส่งคำสั่งที่ชัดเจน ครบถ้วน และถูกต้องให้ AutoNAP

8.4 **Accurate data:** รับผิดชอบต่อความถูกต้องของข้อมูลที่ส่งมาประมวลผล

8.5 **ThaiID authorization:** หากใช้ ThaiID login ผู้ใช้งานยืนยันการอนุญาตทุกครั้ง ลูกค้ารับทราบว่านี่ถือเป็น explicit consent

---

## 9. การตรวจสอบ (Audit)

9.1 ลูกค้ามีสิทธิ์ตรวจสอบ AutoNAP เกี่ยวกับการปฏิบัติตาม DPA ปีละไม่เกิน 1 ครั้ง โดยแจ้งล่วงหน้า 30 วัน

9.2 การตรวจสอบสามารถทำโดย:
- ลูกค้าเอง (on-site หรือ remote)
- ผู้ตรวจสอบอิสระที่ทั้งสองฝ่ายเห็นชอบ
- การรับรองจากบุคคลที่สาม (เช่น ISO 27001, SOC 2) — AutoNAP สามารถส่งรายงานแทนการตรวจสอบ on-site

9.3 ค่าใช้จ่ายในการตรวจสอบเป็นของลูกค้า ยกเว้นพบว่า AutoNAP ไม่ปฏิบัติตาม DPA ในสาระสำคัญ

---

## 10. การสิ้นสุดสัญญาและการคืน/ทำลายข้อมูล

10.1 **เมื่อสัญญาสิ้นสุด** (ไม่ว่าเหตุใด) AutoNAP จะดำเนินการตามที่ลูกค้าเลือก ภายใน **30 วัน**:
- (ก) **ส่งคืน** audit log ในรูปแบบ CSV/JSON หรือ
- (ข) **ทำลาย** ข้อมูลทั้งหมด รวม backup

10.2 AutoNAP จะจัดทำ **หนังสือยืนยัน** (certificate of destruction) เมื่อดำเนินการทำลายข้อมูลแล้ว

10.3 **ข้อยกเว้น:** AutoNAP อาจเก็บข้อมูลต่อไปได้หาก:
- มีข้อผูกมัดทางกฎหมายให้เก็บ (เช่น บันทึกการชำระเงินตามกฎหมายภาษี)
- ข้อมูลถูกทำ anonymized แล้วไม่สามารถระบุตัวตนได้

---

## 11. Liability (ความรับผิดชอบ)

11.1 AutoNAP รับผิดเฉพาะความเสียหายที่เกิดจากการ**ละเมิด DPA โดยตรง** ไม่รวมความเสียหายที่เกิดจาก:
- คำสั่งที่ไม่ถูกต้องจากลูกค้า
- ข้อมูลที่ไม่ถูกต้องจากลูกค้า
- การกระทำของ sub-processor ที่ลูกค้าอนุมัติ
- เหตุสุดวิสัย (force majeure)

11.2 **ขอบเขตความรับผิด:** ความรับผิดโดยรวมของ AutoNAP ในหนึ่งปีไม่เกิน**จำนวนเงินค่าบริการที่ได้รับจากลูกค้าในปีนั้น**

11.3 การจำกัดความรับผิดนี้ไม่รวมถึง:
- การกระทำโดยจงใจ (gross negligence / wilful misconduct)
- การละเมิด PDPA ที่ทำให้เกิดความเสียหายต่อ data subject

---

## 12. การแก้ไข DPA

DPA นี้อาจแก้ไขได้โดยความเห็นชอบเป็นลายลักษณ์อักษรจากทั้งสองฝ่าย หากมีการเปลี่ยนแปลงกฎหมายที่มีนัยสำคัญ AutoNAP จะแจ้งลูกค้าและเสนอการแก้ไข DPA ให้สอดคล้อง

---

## 13. กฎหมายและเขตอำนาจ

13.1 DPA นี้อยู่ภายใต้บังคับของกฎหมายไทย

## Appendix A — Technical and Organizational Measures (TOMs)

### Physical security
- Server อยู่ใน data center ที่ได้รับรองมาตรฐาน (ISO 27001 หรือเทียบเท่า) ในประเทศไทย
- การเข้าถึง physical ต้องมี biometric / key card

### Network security
- Firewall + IDS/IPS
- DDoS protection
- TLS 1.2+ สำหรับทุก connection
- VPN สำหรับ admin access

### Application security
- Authentication: ThaiID / credentials (encrypted at rest)
- Authorization: role-based access control
- Input validation + output encoding
- Regular security scanning (SAST/DAST)

### Data security
- Encryption at rest: AES-256
- Encryption in transit: TLS 1.2+
- Backup: encrypted, stored separately
- **Data minimization: เก็บเฉพาะ masked PID + NAP code + timestamp**
- **Auto-deletion: 90 days retention for audit log**

### Incident response
- Monitoring 24/7
- Incident response plan (documented)
- Post-incident review

### Personnel security
- Background check สำหรับพนักงานที่เข้าถึงข้อมูล
- NDA เซ็นกับพนักงานทุกคน
- PDPA training ประจำปี
- Least-privilege access
