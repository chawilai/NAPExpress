## 🎯 สรุปสั้น (TL;DR)

AutoNAP **ไม่เก็บข้อมูลผู้รับบริการของคลินิก** ที่ท่าน upload มาประมวลผล เก็บไว้เฉพาะ:

- ✅ **เลข 4 ตัวท้าย**ของเลขประจำตัวประชาชน (xxxx1234)
- ✅ **รหัส NAP** ที่ได้จากระบบ สปสช.
- ✅ **เวลาและสถานะ**การประมวลผล

**ไม่เก็บ:** ชื่อ-นามสกุล, เบอร์โทร, ที่อยู่, อีเมล, ข้อมูลการแพทย์, ข้อมูลรายละเอียดอื่น

ข้อมูล audit ข้างต้นถูก**ลบอัตโนมัติหลัง 90 วัน**

- **ชื่อนิติบุคคล:** [ระบุ]
- **ที่ตั้ง:** [ระบุ] (ประเทศไทย)
- **อีเมลติดต่อ:** [ระบุ]
- **DPO (Data Protection Officer):** [ระบุชื่อ + email]

### 1.2 บทบาทของเรา

เมื่อท่านใช้บริการ AutoNAP:
- **ในฐานะผู้ประมวลผล (Data Processor):** สำหรับข้อมูลผู้รับบริการของคลินิก (ปลายทางคือ NAP Plus)
- **ในฐานะผู้ควบคุมข้อมูล (Data Controller):** สำหรับข้อมูลบัญชีและการใช้งานของลูกค้า (คลินิก/มูลนิธิ) ที่สมัครใช้บริการ

## 2. ข้อมูลที่เราเก็บ

### 2.1 ข้อมูลผู้รับบริการของคลินิก (End users ของคลินิก)

**📌 หลักการสำคัญ:** เราใช้ "Process-then-forget" — ประมวลผลแล้วลบทิ้ง

#### 🔄 ข้อมูลที่ประมวลผลชั่วคราว (In-memory เท่านั้น)

ในระหว่างการประมวลผลแต่ละ job เรา**เข้าถึงข้อมูลเหล่านี้ชั่วคราว** เพื่อส่งเข้าระบบ NAP Plus:

- เลขประจำตัวประชาชน 13 หลัก
- UIC (รหัสผู้รับบริการของคลินิก)
- วันที่ให้บริการ
- Key Population (KP)
- อาชีพ
- ข้อมูลอื่นในฟอร์ม NAP

**ข้อมูลเหล่านี้ถูกลบออกจากระบบทันทีหลังประมวลผลเสร็จสิ้น** และไม่เก็บใน database

#### 💾 ข้อมูลที่เก็บรักษา (Audit log)

เราเก็บเฉพาะข้อมูลต่อไปนี้เพื่อให้ท่านสามารถตรวจสอบย้อนหลังได้:

| ฟิลด์ | ตัวอย่าง |
|---|---|
| PID masked (4 ตัวท้าย) | `xxxx1234` |
| NAP code | `RR-2569-123456` |
| Form type | `RR` / `VCT` |
| Status | `success` / `failed` |
| Timestamp | `2026-04-15 10:30:15` |

**ระยะเวลาเก็บ: 90 วัน** หลังจากนั้นระบบจะลบอัตโนมัติ

#### ❌ ข้อมูลที่เราไม่เก็บ

- ชื่อ-นามสกุล
- เลข PID เต็ม 13 หลัก
- เบอร์โทรศัพท์
- อีเมล
- ที่อยู่
- ประวัติการรักษา
- ผลตรวจทางการแพทย์รายละเอียด
- CSV/Excel ไฟล์ที่ upload (ลบหลังประมวลผล)

### 2.2 ข้อมูลบัญชีของลูกค้า (คลินิก/มูลนิธิที่สมัครใช้บริการ)

เพื่อให้บริการเราเก็บ:

- ชื่อคลินิก/มูลนิธิ
- ชื่อผู้ติดต่อ
- อีเมล
- เบอร์โทรศัพท์
- ข้อมูลสำหรับออกใบเสร็จ (ที่อยู่, เลขผู้เสียภาษี)
- ข้อมูลการเข้าใช้งานระบบ (login log)

**ระยะเวลาเก็บ:** ตลอดระยะเวลาสัญญา + 5 ปี ตามกฎหมายภาษี

### 2.3 ข้อมูลการใช้งาน

- จำนวนเคสที่ประมวลผล
- สถิติการใช้งานแบบรวม (aggregate, ไม่ระบุตัวบุคคล)
- IP address + user agent (สำหรับ security log)

### 4.1 ข้อมูลผู้รับบริการ

### 4.2 ข้อมูลบัญชีลูกค้า
- ให้บริการและจัดการบัญชี
- ออกใบเสร็จและใบแจ้งหนี้
- สนับสนุนลูกค้า (customer support)
- แจ้งข่าวสารที่เกี่ยวข้องกับบริการ (เช่น update, maintenance)

## 6. การแชร์ข้อมูลกับบุคคลที่สาม

เราแชร์ข้อมูลเฉพาะในกรณีต่อไปนี้เท่านั้น:

### 6.1 NHSO (สปสช.)
เราส่งข้อมูลเคสเข้าระบบ **NAP Plus ของ สปสช.** — นี่คือวัตถุประสงค์หลักของบริการ

### 6.2 Sub-Processors
เราใช้ผู้ให้บริการ infrastructure ที่จำเป็น (ทั้งหมดตั้งอยู่ในประเทศไทย หรือเฉพาะ metadata ที่ไม่มี PII):

| ผู้ให้บริการ | หน้าที่ | ที่ตั้งข้อมูล |
|---|---|---|
| VPS provider | Server hosting | ไทย |
| Email service (SMTP) | ส่ง report | ไทย/ASEAN |
| Realtime service (Ably) | Progress updates | Global CDN (ไม่มี PII — แค่ status code) |

ผู้ให้บริการทั้งหมดมีข้อตกลง DPA ที่มีมาตรฐานไม่ต่ำกว่านโยบายนี้

### 6.3 หน่วยงานของรัฐ
เราอาจเปิดเผยข้อมูลตามคำสั่งศาลหรือกฎหมาย — ในกรณีเช่นนั้น เราจะแจ้งท่านล่วงหน้าหากกฎหมายอนุญาต

## 7. ที่ตั้งของข้อมูล (Data Localization)

ข้อมูลทั้งหมดของ AutoNAP **จัดเก็บบน server ในประเทศไทย** เราไม่ถ่ายโอนข้อมูลออกนอกประเทศโดยไม่ได้รับความยินยอม

ข้อยกเว้น: metadata เล็กน้อยของ realtime service (Ably) อาจผ่าน CDN ระดับโลก แต่เป็นข้อมูลที่**ไม่มี PII** (เช่น progress percentage, status code)

## 9. สิทธิ์ของเจ้าของข้อมูล (Data Subject Rights)

ตาม PDPA มาตรา 30-37 ท่านมีสิทธิ์ดังต่อไปนี้:

| สิทธิ์ | รายละเอียด | วิธีใช้สิทธิ์ |
|---|---|---|
| **Right of Access** | ขอดูข้อมูลที่เรามี | ส่งคำขอมาที่ privacy@autonap.co.th |
| **Right to Rectification** | ขอแก้ไขข้อมูลที่ผิด | - |
| **Right to Erasure** | ขอให้ลบข้อมูล | เราลบภายใน 7 วันทำการ |
| **Right to Restrict** | ขอระงับการประมวลผล | - |
| **Right to Data Portability** | ขอ export ข้อมูลในรูปแบบ CSV/JSON | - |
| **Right to Object** | คัดค้านการประมวลผล | - |
| **Right to Withdraw Consent** | ถอนความยินยอม | ยกเลิกบัญชีได้ทุกเมื่อ |
| **Right to Complain** | ร้องเรียนต่อ PDPC | ติดต่อ PDPC ที่ [pdpc.or.th](https://www.pdpc.or.th) |

**หมายเหตุ:** สำหรับผู้รับบริการของคลินิก (data subject) สิทธิ์เหล่านี้ใช้กับ data controller (คลินิกที่รับบริการ) โดยตรง เพราะเราเป็นเพียง processor AutoNAP จะช่วย controller ในการตอบสนองคำขอของ data subject

## 10. Cookies และ Tracking

### 10.1 Cookies ที่เราใช้

- **Essential cookies:** จำเป็นสำหรับการทำงานของระบบ (session, authentication) — ไม่ต้องขอความยินยอม
- **Analytics cookies:** (optional) เก็บสถิติการใช้งาน อาจใช้ Plausible (privacy-friendly, ไม่มี PII) — ขอความยินยอมก่อนใช้

### 10.2 เราไม่ใช้

- ❌ Google Analytics (GA4)
- ❌ Facebook Pixel
- ❌ Third-party advertising cookies
- ❌ Cross-site tracking
