# คู่มือ: ระบบยืนยันและพิสูจน์ตัวตน (Authentication) ของ สปสช. สำหรับหน่วยบริการ

> เอกสารฉบับนี้เป็นเอกสารอ้างอิงภายในของโปรเจกต์ AutoNAP จัดทำขึ้นเพื่อให้ทีมและลูกค้า (คลินิก/NGO ที่ทำ HIV/STI/HCV prevention) เข้าใจว่าระบบ "authen" ของ สปสช. คืออะไร ขอสิทธิได้อย่างไร ใช้งานอย่างไร และมีโอกาส integrate กับ AutoNAP ได้แค่ไหน
>
> **วันที่เรียบเรียง:** เมษายน 2569
> **คำเตือน:** ข้อมูลสิทธิประโยชน์ อัตราเบิกจ่าย และขั้นตอนการสมัคร สปสช. มีการปรับเปลี่ยนบ่อย (เคยมีการปรับครั้งใหญ่ในเดือน ม.ค. 2569) ก่อนดำเนินการจริงให้โทรตรวจสอบกับสายด่วน **1330 กด 5** (บริการสำหรับหน่วยบริการ) หรือติดต่อ สปสช. เขตพื้นที่ก่อนทุกครั้ง

---

## 1. บทนำ — ระบบ authen ของ สปสช. คืออะไร

**Authentication** (หรือที่เรียกว่า "ระบบยืนยันและพิสูจน์ตัวตนผู้รับบริการ" / "Authen Code" / "New Authen") คือกระบวนการที่ สปสช. กำหนดให้หน่วยบริการต้อง **ยืนยันว่าผู้รับบริการมาถึงหน่วยจริง** ก่อนจะบันทึกข้อมูลและเบิกค่าบริการจาก สปสช. วัตถุประสงค์:

1. ยืนยันตัวตนผู้รับบริการว่าเป็นเจ้าของสิทธิจริง (กันทุจริต/สวมสิทธิ)
2. เป็นหลักฐานอ้างอิงประกอบการเบิกจ่ายค่ารักษาพยาบาล/ค่าบริการ
3. ผูกข้อมูลบริการเข้ากับเลขบัตรประชาชนเพื่อส่งเคลมอัตโนมัติ

ปัจจุบัน สปสช. มีวิธียืนยันตัวตนอย่างน้อย **4 ระบบ** ([สปสช., news/3865](https://www.nhso.go.th/news/3865), [Bangkok Biz News](https://www.bangkokbiznews.com/health/public-health/1046845)):

| ลำดับ | ระบบ | วิธียืนยัน | เหมาะกับ |
| --- | --- | --- | --- |
| 1 | **ERM** | เสียบบัตรประชาชน smart card เข้าเครื่องอ่านบัตร (หรือเสียบบัตรของผู้ปกครอง/ผู้รับรองแทน) | รพ./หน่วยที่ใช้ระบบ ERM เดิม |
| 2 | **LINE Official Account @nhso** | หน่วยบริการแสดง QR code ให้ผู้รับบริการสแกนผ่าน LINE เพื่อยืนยันตัวเอง | ผู้รับบริการที่มี smartphone |
| 3 | **New Authen (New Authen Code Online)** | เสียบบัตร smart card **หรือ** ถ่ายรูปผู้ป่วยคู่กับบัตรประชาชน **หรือ** ให้ผู้ปกครอง/ผู้รับรองยืนยันแทน | หน่วยบริการทั่วไปรวมถึงคลินิก, NGO, ร้านยา (ใช้ได้แพร่หลายที่สุดในปัจจุบัน) |
| 4 | **ตู้คีออส / Station** | เสียบบัตรประชาชนเข้าเครื่องอ่านบัตรที่ตู้ตั้งเอง | รพ./หน่วยที่มีจุดบริการตู้ |

สำหรับคลินิก NGO ขนาดเล็กที่ทำ HIV prevention ระบบที่ใช้งานได้จริงและมีค่าใช้จ่ายต่ำที่สุดคือ **New Authen Online** โดยใช้คอมพิวเตอร์ที่มีอยู่เดิม ต่อเครื่องอ่านบัตรสมาร์ทการ์ดและ/หรือกล้อง webcam ([สปสช. news/3842 "DIY คอมพิวเตอร์ยืนยันตัวตน"](https://www.hfocus.org/content/2022/12/26585))

## 2. ใครขอสิทธิ authen ได้ (Eligibility)

การจะใช้ระบบ authen ของ สปสช. ได้ หน่วยงานต้อง **ขึ้นทะเบียนเป็น "หน่วยบริการ" ในระบบหลักประกันสุขภาพแห่งชาติก่อน** เมื่อขึ้นทะเบียนแล้ว สปสช. จึงจะออก username/password สำหรับเข้าระบบ New Authen Code Online และระบบงานชดเชยอื่น ๆ ให้

ประเภทหน่วยบริการที่ **เกี่ยวข้องกับงาน HIV/STI/HCV prevention** มี 2 หมวดหลัก:

### 2.1 หน่วยบริการนวัตกรรม (Innovation Service Unit)
สปสช. เปิดรับสมัคร **คลินิก 7 ประเภท + ร้านยา** ให้เป็นหน่วยบริการนวัตกรรมเพื่อรองรับนโยบาย "30 บาทรักษาทุกที่" ([Hfocus 30633](https://www.hfocus.org/content/2024/05/30633), [Hfocus 36716](https://www.hfocus.org/content/2026/01/36716)):

1. คลินิกเวชกรรม
2. คลินิกการพยาบาลและการผดุงครรภ์
3. คลินิกทันตกรรม
4. คลินิกเทคนิคการแพทย์
5. คลินิกการแพทย์แผนไทย
6. คลินิกกายภาพบำบัด
7. คลินิก (ประเภทอื่นตามประกาศเพิ่มเติม)
8. ร้านยา GPP / ร้านยาคุณภาพ

หน่วยบริการนวัตกรรม **ไม่ต้องวางหลักประกันสัญญา** และ สปสช. ประกาศขึ้นทะเบียน+ลงนามนิติกรรมภายใน 5 วันทำการ หลังเอกสารครบ

### 2.2 หน่วยบริการรับส่งต่อเฉพาะด้าน — "หน่วยบริการ มาตรา 3"
นี่คือช่องทางสำคัญสำหรับ **NGO/องค์กรภาคประชาสังคม** ที่ทำงานด้าน HIV โดยเฉพาะ ตามมาตรา 3 แห่ง พ.ร.บ. หลักประกันสุขภาพแห่งชาติ ปัจจุบัน สปสช. ขยาย "หน่วยบริการรับส่งต่อเฉพาะด้าน" ให้ครอบคลุม **8 ด้าน** รวมถึง (1) บริการเอชไอวี (2) อนามัยเจริญพันธุ์ (3) ยาเสพติด ฯลฯ ([Matichon](https://www.matichon.co.th/local/quality-life/news_4720827), [HON](https://hon.co.th/health-service-center/), [Hfocus 35934](https://www.hfocus.org/content/2025/11/35934))

**คุณสมบัติหลักในการขึ้นทะเบียนมาตรา 3 (องค์กรภาคประชาสังคม):**
- เป็นองค์กรที่มีประสบการณ์ให้บริการด้านที่ขอไม่น้อยกว่า **1 ปี**
- ได้รับการรับรองมาตรฐานจากหน่วยงานกำกับดูแลที่เกี่ยวข้อง (เช่น กรมควบคุมโรค, กองโรคเอดส์ฯ)
- มีบุคลากรและสถานที่ให้บริการตามเกณฑ์ที่ สปสช. กำหนด

ศูนย์องค์รวม / ศอร. / ศูนย์บริการชุมชนของเครือข่าย ผู้ติดเชื้อเอชไอวี/เอดส์, IHRI, มูลนิธิแคร์แมท, มูลนิธิรักษ์ไทย, สกายสีม่วงพัทลุง, บ้าน HON ฯลฯ ได้ขึ้นทะเบียนแนวนี้แล้ว ปัจจุบันมีองค์กรภาคประชาสังคมร่วมเป็น "หน่วยบริการรับส่งต่อเฉพาะด้าน HIV/AIDS" กว่า **71 องค์กร** ([สปสช. news/3832](https://www.nhso.go.th/news/3832), [สปสช. news/4080](https://www.nhso.go.th/news/4080))

### 2.3 รพ./รพ.สต. ในระบบ
รพ. รัฐและเอกชนที่ทำสัญญากับ สปสช. อยู่แล้ว จะได้สิทธิ authen โดยอัตโนมัติไม่ต้องขอแยก

## 3. ขั้นตอนการสมัคร (Checklist)

### 3.1 ช่องทางการสมัครหลัก
- **ระบบออนไลน์ One Stop Service** ที่ [https://ossregister.nhso.go.th](https://ossregister.nhso.go.th) ([Hfocus 28724](https://www.hfocus.org/content/2023/10/28724))
- **ระบบข้อมูลพื้นฐานหน่วยบริการ** ที่ [https://cpp.nhso.go.th](https://cpp.nhso.go.th/)
- **สปสช. เขตพื้นที่** (เขต 1–13) ให้คำปรึกษาและตรวจเอกสาร
- **สายด่วน สปสช. 1330 กด 5** สำหรับหน่วยบริการ ([สปสช. news/3810](https://www.nhso.go.th/news/3810))

### 3.2 เอกสารที่ต้องเตรียม (แนวทั่วไป — กรุณายืนยันกับเขตพื้นที่ก่อนยื่น)
สำหรับ **คลินิก/ร้านยา** (หน่วยบริการนวัตกรรม):
- [ ] สำเนาใบอนุญาตประกอบกิจการสถานพยาบาล (สพ.) / ใบอนุญาตร้านยา
- [ ] สำเนาใบประกอบวิชาชีพของผู้ดำเนินการ
- [ ] ทะเบียนบ้านสถานประกอบการ / หนังสือรับรองนิติบุคคล (ถ้ามี)
- [ ] สำเนาบัตรประชาชนผู้ดำเนินการ
- [ ] แผนที่/พิกัดที่ตั้ง
- [ ] แบบคำขอขึ้นทะเบียน (มีให้ในระบบ OSS)
- [ ] ข้อมูลบัญชีธนาคารสำหรับรับเงินเบิกจ่าย

สำหรับ **NGO/ภาคประชาสังคม** (หน่วยบริการ มาตรา 3):
- [ ] หนังสือจดทะเบียนมูลนิธิ/สมาคม/องค์กรสาธารณประโยชน์
- [ ] เอกสารแสดงประสบการณ์ให้บริการ HIV/STI ไม่น้อยกว่า 1 ปี (เช่น รายงานโครงการ RRTTR, PRIDE, สัญญาทุน Global Fund/สสส./กรมควบคุมโรค)
- [ ] หนังสือรับรองมาตรฐานจากกรมควบคุมโรค/กองโรคเอดส์ฯ (ถ้ามี)
- [ ] รายชื่อและคุณสมบัติบุคลากร (พยาบาล, lay provider ที่ผ่านอบรม)
- [ ] แผนผังสถานที่และอุปกรณ์
- [ ] ข้อมูลเครือข่ายรับส่งต่อ (refer) กับหน่วยบริการหลัก เช่น รพ.ที่เป็นคู่สัญญา
- [ ] บัญชีธนาคารขององค์กร

### 3.3 ระยะเวลาพิจารณา
- หน่วยบริการนวัตกรรม (OSS): ประมาณ **5 วันทำการ** หลังเอกสารครบ
- หน่วยบริการ มาตรา 3: หลายสัปดาห์ — หลายเดือน เพราะต้องผ่านการประเมินจากคณะทำงานและกรมควบคุมโรค ([สปสช. news/3911](https://www.nhso.go.th/news/3911))

### 3.4 ค่าใช้จ่าย
- **ไม่มีค่าธรรมเนียม** การสมัคร สปสช. โดยตรง
- **ไม่ต้องวางหลักประกันสัญญา** (สำหรับหน่วยบริการนวัตกรรม)
- ค่าใช้จ่ายเกิดกับ **อุปกรณ์** (ดูหัวข้อ 4) และ **เวลา/คนทำเอกสาร**

### 3.5 หลังได้รับอนุมัติ
1. สปสช. ออก **รหัสหน่วยบริการ** (hcode — จาก [hcode.moph.go.th](https://hcode.moph.go.th))
2. สปสช. ออก **username/password** สำหรับเข้าระบบงานชดเชย, New Authen, NAP Plus และระบบอื่นที่เกี่ยวข้อง
3. หน่วยเข้าร่วมการอบรมการใช้งาน (จัดโดย สปสช. เขตพื้นที่)

## 4. อุปกรณ์และ Software ที่ต้องมี

### 4.1 Hardware (ขั้นต่ำสำหรับ New Authen Online)
- **คอมพิวเตอร์** PC หรือ Notebook ระบบปฏิบัติการ Windows (แนะนำ Windows 10/11)
- **อินเทอร์เน็ต** ที่เสถียร
- **เครื่องอ่านบัตรสมาร์ทการ์ด (Smart Card Reader)** รุ่นที่รองรับบัตรประชาชนไทย (เช่น ACS ACR38/39, Feitian, SCR3310) — ราคาหลักร้อยบาท
- **กล้อง Webcam** (ใช้กรณีถ่ายรูปผู้ป่วย+บัตร เป็นทางเลือกแทน smart card)
- **เครื่องพิมพ์** (optional — ถ้าต้องพิมพ์ใบ authen หรือหลักฐานให้ผู้รับบริการ)

สปสช. ยืนยันว่าอุปกรณ์เหล่านี้ **ราคาไม่แพง** และหน่วยบริการสามารถ "DIY" ขึ้นเองได้ ([Hfocus 26585](https://www.hfocus.org/content/2022/12/26585))

### 4.2 Software ที่ต้องติดตั้ง
1. **E-Form Agent** — โปรแกรมตัวกลางที่อ่านข้อมูลจาก smart card reader แล้วส่งเข้าเว็บ สปสช. ดาวน์โหลดจาก [https://www.nhso.go.th/downloads/68](https://www.nhso.go.th/downloads/68) ติดตั้งและ restart เครื่องก่อนใช้งาน ([สปสช. news/3842](https://www.nhso.go.th/news/3842))
2. **Web Browser** สมัยใหม่ (Chrome / Edge แนะนำ) — ระบบ New Authen ใช้งานผ่านเว็บ
3. **Driver ของเครื่องอ่านบัตร** ตามยี่ห้อที่ซื้อ
4. (ทางเลือก) **แอป Authen By NHSO** บน Android/iOS — อีกช่องทางสำหรับยืนยันตัวตน ([Google Play](https://play.google.com/store/apps/details?id=th.go.nhso.authenbynhso))
5. (ทางเลือก) **ThaID** — แอปของกรมการปกครองสำหรับ Digital ID อาจใช้ประกอบในบาง flow

### 4.3 เข้าระบบผ่าน
- เว็บไซต์ สปสช. → เมนู **"บริการออนไลน์" → "งานชดเชย" → "ระบบตรวจสอบและยืนยันการเข้ารับบริการ New Authen Code"**
- หรือเข้าตรงที่ [https://authenservice.nhso.go.th/authencode/](https://authenservice.nhso.go.th/authencode/) (redirect ผ่าน Keycloak OAuth2 ของ สปสช.)
- Login ด้วย **username/password** ที่ สปสช. ออกให้ตอนขึ้นทะเบียน (ใช้ระบบ IAM/Keycloak เดียวกับระบบอื่นของ สปสช.)

## 5. การใช้งานจริง (Workflow)

ขั้นตอนปกติของการใช้ระบบ New Authen Code เพื่อบันทึกการให้บริการหนึ่งครั้ง:

1. **ผู้รับบริการมาถึงหน่วย** → ยื่นบัตรประชาชน
2. เจ้าหน้าที่ **login** เข้าระบบ New Authen Code Online ด้วย username/password
3. **เสียบบัตร smart card** ของผู้รับบริการเข้าเครื่องอ่านบัตร (หรือเลือก "ถ่ายรูปผู้ป่วย+บัตร" กรณีไม่มีบัตร smart card หรือเครื่องอ่านขัดข้อง)
4. E-Form Agent จะส่งข้อมูลบัตรเข้าระบบ → หน้าจอแสดงชื่อ, เลขบัตร, สิทธิการรักษา (บัตรทอง/ประกันสังคม/ข้าราชการ/สิทธิว่าง)
5. เจ้าหน้าที่ **ตรวจสอบ** ข้อมูลบนหน้าจอตรงกับผู้รับบริการตัวจริง → กดยืนยัน
6. ระบบสร้าง **Authen Code** (รหัสอ้างอิงการยืนยันตัวตนครั้งนั้น)
7. เจ้าหน้าที่ **เลือกรายการบริการ** ที่จะให้ (เช่น HIV self-test, HCV screening, PrEP dispensing, ปรึกษา) ตามชุดสิทธิประโยชน์ที่หน่วยตนได้รับอนุมัติ
8. บันทึกข้อมูลบริการ → ระบบเก็บ Authen Code + รหัสบริการเข้าฐานข้อมูล สปสช.
9. สำหรับงาน HIV เฉพาะ: ข้อมูลบางส่วนต้องบันทึกซ้ำใน **NAP Plus** (ระบบที่ AutoNAP จัดการอยู่) เพราะ NAP เป็นระบบเฉพาะของกลุ่มโรคเอดส์/วัณโรค ไม่ใช่ระบบเดียวกับ New Authen
10. สปสช. ประมวลผลเคลมตามรอบ (รายสัปดาห์/รายเดือน) → โอนเงินเข้าบัญชีหน่วย

> **หมายเหตุสำคัญ:** ตามข้อมูลที่หาได้ ยังไม่ชัดเจนว่าระบบ New Authen เชื่อมกับ NAP Plus แบบ seamless หรือไม่ หน่วยบริการส่วนใหญ่ยังต้องบันทึกข้อมูลสองที่ (authen + NAP) — ต้องยืนยันกับ สปสช. เขตและทีมงาน NAP Plus

## 6. บริการที่เลือกได้ผ่านระบบ (เน้น HIV/STI/HCV)

ชุดสิทธิประโยชน์ที่บันทึกผ่านระบบงานชดเชย/authen ได้ มีหลายรายการ สำหรับลูกค้า AutoNAP ที่ทำ HIV prevention รายการสำคัญคือ:

### 6.1 HIV Self-Testing (HIVST)
- **ผู้ได้รับสิทธิ:** คนไทยทุกสิทธิ (บัตรทอง, ประกันสังคม, ข้าราชการ)
- **เงื่อนไข:** ผู้เกิดก่อน พ.ศ. 2535 รับได้ 1 ครั้งตลอดชีวิต; กลุ่มเสี่ยง 5 กลุ่ม รับได้ 1 ครั้ง/ปี; ไม่เกิน 1 test/วัน
- **หน่วยที่ให้บริการได้:** รพ., คลินิกเวชกรรม, คลินิกพยาบาล, คลินิกเทคนิคการแพทย์, ร้านยา, ศูนย์บริการ NGO ที่ขึ้นทะเบียน ([TheCoverage 6810](https://www.thecoverage.info/news/content/6810), [TheCoverage 5390](https://www.thecoverage.info/news/content/5390))
- **อัตราเบิกจ่าย:** ~**100 บาทต่อครั้ง** (รวมค่าชุดตรวจและค่าให้คำปรึกษา — ยืนยันกับ สปสช. เขต เพราะอัตราอาจปรับ)
- **ระบบลงทะเบียน:** [https://hivsst.ddc.moph.go.th](https://hivsst.ddc.moph.go.th) (กรมควบคุมโรค) + ยืนยันสิทธิผ่าน New Authen

### 6.2 HIV Confirmatory Testing / VCT
- ตรวจยืนยันหลังจาก HIVST ผล reactive
- ต้องทำที่ รพ./คลินิกที่มีห้องแล็บ
- บันทึกใน NAP Plus (ระบบที่ AutoNAP จัดการ)

### 6.3 PrEP (Pre-Exposure Prophylaxis)
- ยาป้องกัน HIV ก่อนสัมผัส — ฟรีภายใต้สิทธิบัตรทอง ([Love Foundation](https://lovefoundation.or.th/en/nhso-provides-prep/))
- ต้องผ่านการประเมินกลุ่มเสี่ยงและตรวจ HIV ก่อนจ่ายยา
- หน่วยที่จ่ายได้: รพ., คลินิกเฉพาะทาง, หน่วยบริการ มาตรา 3 ที่ได้รับอนุมัติ ([Buddy Station](http://buddystation.ddc.moph.go.th/category/clinic))

### 6.4 PEP (Post-Exposure Prophylaxis)
- ยาป้องกันหลังสัมผัสเชื้อ (ภายใน 72 ชม.)
- เบิกฟรีผ่านสิทธิบัตรทอง

### 6.5 STI Screening & Treatment
- ตรวจซิฟิลิส, หนองใน, คลามีเดีย, HPV ฯลฯ
- ฟรีสำหรับคนไทยทุกสิทธิ

### 6.6 Hepatitis C (HCV) Testing & Treatment
- ตรวจคัดกรอง HCV ฟรีในกลุ่มเสี่ยง (PWID, PLHIV, MSM, ผู้ต้องขัง ฯลฯ)
- ยารักษา DAA (Direct-Acting Antivirals) อยู่ในชุดสิทธิประโยชน์บัตรทอง

### 6.7 ถุงยางอนามัย / Condom
- แจกฟรีผ่านหน่วยบริการที่ขึ้นทะเบียน

### 6.8 อื่น ๆ ที่อาจเกี่ยวข้อง
- วัคซีน (HPV, Hep B)
- คัดกรอง TB
- บริการอนามัยเจริญพันธุ์ (ยาคุม, IUD)

> **ข้อควรระวัง:** รายการที่หน่วยบริการของคุณสามารถเลือกบันทึก **ขึ้นกับประเภทใบอนุญาต + ข้อตกลงกับ สปสช. เขต** ไม่ใช่ทุกหน่วยจะให้บริการได้ทุกรายการ — ต้องเช็คที่ สปสช. เขตว่า hcode ของคุณถูก "เปิดสิทธิ" ให้บันทึกรายการใดบ้าง

## 7. การเบิกจ่าย (Reimbursement)

- **ช่องทาง:** หน่วยบันทึกข้อมูลผ่านระบบงานชดเชยของ สปสช. → สปสช. ประมวลผลตามรอบ → โอนเงินเข้าบัญชี
- **รอบเบิกจ่าย:** ปกติรายเดือน (บาง fee-schedule รายสัปดาห์) — ระบบใหม่ที่เริ่ม 12 ม.ค. 2569 มีการจัดโควตารายสัปดาห์สำหรับหน่วยนวัตกรรม ([TheCoverage 10417](https://www.thecoverage.info/news/content/10417), [Hfocus 36716](https://www.hfocus.org/content/2026/01/36716))
- **อัตราจ่าย:** แตกต่างกันตามรายการ เช่น HIVST ~100 บาท/ครั้ง, ตรวจ HIV confirmatory, ยา PrEP ฯลฯ มีประกาศแนบท้ายของ สปสช. เป็นปี ๆ — ให้อ้างอิงประกาศแนวทางการเบิกจ่ายประจำปีของ สปสช.
- **ความต่างจากการบันทึก NAP:** NAP Plus เป็นระบบ epidemiological surveillance + reporting สำหรับโรคเอดส์ (แยกจากระบบเบิกจ่ายบางส่วน) — หน่วยที่ทำ HIV ส่วนใหญ่ต้อง **บันทึกทั้ง authen (เพื่อเบิก) + NAP (เพื่อรายงาน)** ซึ่งเป็นเหตุผลที่ AutoNAP มีอยู่

## 8. ข้อมูลสำหรับ Integration (สำคัญสำหรับ AutoNAP)

### 8.1 มี API สาธารณะให้ third party ใช้ไหม?
จากการค้นข้อมูลสาธารณะ **ไม่พบหลักฐาน** ว่า สปสช. เปิด REST API ของระบบ New Authen/งานชดเชยให้ third party เรียกใช้ได้โดยตรง ระบบ authen ทั้งหมดใช้งานผ่าน:
- **Web UI** (ระบบ SSO ผ่าน Keycloak — [iam.nhso.go.th](https://iam.nhso.go.th))
- **E-Form Agent** (โปรแกรม desktop ที่ติดตั้งในเครื่องของหน่วย เพื่ออ่าน smart card แล้วโพสต์เข้าเว็บ)
- **Authen By NHSO** (แอป mobile — ใช้งานเฉพาะเจ้าหน้าที่/หรือผู้รับบริการ)

### 8.2 ทางเลือกในการ integrate กับ AutoNAP
1. **Headless browser automation** (เหมือนที่ AutoNAP ใช้กับ NAP Plus ปัจจุบัน) — Playwright/Puppeteer login เข้า authen service, simulate การเสียบบัตร/ยืนยัน **ติดข้อจำกัด:** E-Form Agent ต้อง run บนเครื่องจริง + ต้องมี smart card จริง ไม่สามารถ simulate จาก server ได้ง่าย ๆ
2. **Smart card reader SDK** — เขียน native integration กับ PC/SC (Windows WinSCard API, pcsclite บน Linux/Mac) เพื่ออ่านบัตรประชาชนไทย (format TIS 620-3, MOI schema) โดยตรง จากนั้นส่ง payload ให้ AutoNAP worker ที่อยู่บน cloud → ยังต้อง reverse-engineer payload ของ authen service
3. **Local agent แบบ Electron** — แพ็ค AutoNAP worker + smart card driver เป็น desktop app ที่ติดตั้งที่หน่วยบริการ ทำงานเป็น "sidecar" ข้าง E-Form Agent
4. **ติดต่อ สปสช. สำนักไอที** เพื่อขอเข้าโปรแกรมพันธมิตรสำหรับ HIS vendor — บริษัท HIS เช่น HOSxP, HAOS น่าจะเคยเจรจามาก่อน อาจมี API ส่วนตัว
5. **ใช้ hook ที่ NAP Plus เปิดไว้** (ถ้ามี) — NAP Plus อาจมีช่องทาง import batch ที่อ้าง Authen Code ที่ได้จากระบบ New Authen แล้ว — ต้องสอบถามทีมงาน NAP Plus

### 8.3 ข้อเสนอแนะสำหรับ AutoNAP
- **ระยะสั้น:** ลูกค้าที่ขอสิทธิ authen ได้แล้วยังใช้ New Authen ด้วยตัวเองผ่าน web UI, แล้วใช้ AutoNAP สำหรับบันทึก NAP Plus ตามเดิม (two-step workflow)
- **ระยะกลาง:** พัฒนา "AutoNAP Desktop Companion" ตัวเล็กที่ทำงานเป็น local agent (อ่าน smart card, ยิง authen, จากนั้นส่ง event ต่อเข้า AutoNAP server เพื่อบันทึก NAP)
- **ระยะยาว:** ติดต่อ สปสช. สำนักเทคโนโลยีสารสนเทศ (ดูข้อมูลติดต่อในหัวข้อ 9) เพื่อถามถึง partnership / API access

### 8.4 คำถามที่ต้องสอบถาม สปสช. โดยตรง
1. มี REST API สำหรับ New Authen Code ให้ vendor ใช้ไหม?
2. Authen Code ที่ได้จากระบบ New Authen สามารถส่งเข้า NAP Plus ได้แบบ batch ไหม?
3. E-Form Agent มี SDK/CLI หรือใช้ได้เฉพาะผ่าน browser?
4. Bulk verification/authentication สำหรับ outreach ได้หรือไม่ (เพราะ NGO outreach ทำในสนามแบบ real-time ไม่ได้ทุกครั้ง)?
5. อัตราเบิกจ่ายปัจจุบันของ HIVST, PrEP, PEP, HCV testing (ต้องขอประกาศแนวทางการเบิกจ่ายประจำปี)

## 10. สิ่งที่ยังไม่ชัดเจน / ต้อง verify เพิ่ม

รายการต่อไปนี้เป็นข้อที่ **ไม่พบข้อมูลสาธารณะที่ชัดเจน** จำเป็นต้องโทร 1330 กด 5 หรือติดต่อ สปสช. เขตเพื่อยืนยัน:

1. **รายการเอกสารขึ้นทะเบียนแบบ exhaustive** ของ "หน่วยบริการ มาตรา 3" (องค์กรภาคประชาสังคม) — ในเอกสารสาธารณะมีเพียงโครงกว้าง ๆ ไม่มี checklist ทางการ
2. **อัตราเบิกจ่ายล่าสุด** ของ HIVST, PrEP, PEP, HCV testing, condom ในประกาศปี 2569 — ตัวเลข 100 บาท/HIVST เป็นข้อมูลเก่า อาจปรับแล้ว
3. **มี REST API สาธารณะสำหรับ New Authen Code หรือไม่** — จากข้อมูลที่หาได้ **ไม่มี** แต่ไม่ยืนยัน 100% ต้องถามสำนัก IT สปสช. โดยตรง
4. **การเชื่อมข้อมูลระหว่าง New Authen ↔ NAP Plus** — ไม่ชัดเจนว่า Authen Code ที่ได้จากระบบใหม่ถูกส่งเข้า NAP Plus อัตโนมัติหรือต้องกรอกซ้ำ
5. **ระยะเวลาพิจารณา** การสมัครหน่วยบริการ มาตรา 3 ที่แน่นอน (บางแหล่งบอกไม่กี่สัปดาห์ บางแหล่งบอกหลายเดือน ขึ้นกับเขตและประเภทบริการ)
6. **ค่าใช้จ่ายแฝง** เช่น ค่าบำรุงระบบ, ค่าอบรม, ค่าอุปกรณ์ที่ สปสช. แนะนำ — ปกติไม่มี แต่ควรยืนยัน
7. **การขึ้นทะเบียนสำหรับ NGO ที่ไม่เคยทำสัญญากับ สปสช. มาก่อน** — คุณสมบัติ "ประสบการณ์ 1 ปี" คำนวณอย่างไร (สัญญากับใคร, หลักฐานอะไร)
8. **เกณฑ์การได้รับอุปกรณ์ฟรี** — บางโครงการ สปสช./คร. เคยสนับสนุน smart card reader ให้หน่วยบริการ ต้องเช็คว่าโครงการปัจจุบันยังมีหรือไม่
9. **กรอบเวลาส่ง claim / กฎการ reject** — ไม่ครอบคลุมในเอกสารนี้
10. **เอกสารทางการของ "ระบบ 4 แบบ"** (ERM, LINE, New Authen, Kiosk) ว่าแต่ละแบบมีข้อจำกัด/เงื่อนไขการเบิกจ่ายต่างกันอย่างไร — หาได้เฉพาะข้อมูลระดับข่าว

### แนะนำขั้นตอนถัดไปสำหรับทีม AutoNAP

1. **โทร 1330 กด 5 กด 0** ขอคุยเจ้าหน้าที่ สอบถาม (ก) ขั้นตอนการขึ้นทะเบียนสำหรับ NGO HIV, (ข) เอกสารที่ต้องใช้, (ค) ติดต่อสำนัก IT ขอเอกสาร API (ถ้ามี)
2. **ติดต่อ สปสช. เขต** ที่ครอบคลุมพื้นที่ลูกค้า (เขต 1 เชียงใหม่สำหรับแคร์แมท, เขต 13 กทม., ฯลฯ) เพื่อคุยเรื่อง onboarding จริง
3. **ติดต่อ IHRI / กองโรคเอดส์ฯ (กรมควบคุมโรค)** ที่เป็นพี่เลี้ยงหลักของ NGO HIV ในประเทศ — มักมีโครงการพี่เลี้ยงการขึ้นทะเบียน
4. **ติดต่อองค์กรที่ขึ้นทะเบียนมาตรา 3 เรียบร้อยแล้ว** เช่น ศูนย์องค์รวม, สกายสีม่วง, บ้าน HON เพื่อขอแบ่งปันประสบการณ์จริง (lessons learned)
5. **หาคู่มือ PDF ทางการของ New Authen** — เอกสาร [uckkpho.com PDF ปี 2565](https://www.uckkpho.com/wp-content/uploads/2023/03/พชระ-Authentication16032565_06.pdf) น่าจะเป็นจุดตั้งต้นที่ดี ดาวน์โหลดและอ่านฉบับเต็มแบบออฟไลน์
